MITRE ATT&CK + Zero Trust

Using frameworks to explain the attack and justify the controls

This page translates the Sportetitive incident into a structured attack path so the recommended security controls are easier to defend in front of the board. MITRE ATT&CK helps describe how the attack unfolded, while Zero Trust explains the management response: trust less, verify more, reduce privilege and monitor continuously.

Reconnaissance

Attackers used public information, names, roles and supplier relationships to prepare a believable social engineering approach. This highlights a governance issue as much as a technical one, because supplier exposure and publicly visible context increased the chance of a successful pretext.

Zero Trust control response: reduce unnecessary exposure, strengthen supplier assurance and include third-party relationships in cyber risk reporting.

Initial access

A convincing fake survey link was sent through a trusted-looking message. The MSP relationship lowered suspicion and helped the attackers reach the first foothold without needing a complex exploit.

Zero Trust control response: strengthen email filtering, improve phishing awareness, verify requests more carefully and avoid treating supplier-origin traffic as automatically safe.

Credential access

Keylogger activity captured privileged credentials, which then became the main route into the organisation’s environment. This is one of the most important lessons from the incident because a single powerful account acted like a master key.

Zero Trust control response: enforce MFA, separate admin accounts from normal user accounts, reduce standing privilege and review access rights regularly.

Lateral movement

Once privileged access had been captured, the attackers moved across systems, explored the environment and reached services that should have been harder to traverse. This reflects the risk of broad trust across stores, HQ systems, cloud services and supplier-linked access paths.

Zero Trust control response: use segmentation, conditional access, tighter remote service control and stronger visibility over unusual authentication and admin activity.

Exfiltration and impact

Personal data was stolen and systems were later encrypted for extortion and disruption. At this point the event stopped being only an outage problem and became a regulatory, reputational and governance issue.

Zero Trust control response: improve data classification, monitor abnormal transfers, protect backups, define the recovery order in advance and ensure the board is prepared for legal reporting decisions.

CIO interpretation

For the CIO, these frameworks are useful because they convert a technical incident into a management narrative. MITRE ATT&CK helps explain how the attack progressed, while Zero Trust turns that explanation into a control strategy built around identity, segmentation, monitoring, supplier governance, backup resilience and board assurance.

Board-level conclusion

Sportetitive’s ransomware incident was a business crisis, not only an IT failure.

The incident exposed weaknesses in governance, supplier trust, identity management, monitoring, legal readiness and legacy technology. This website brings the core findings into one place so the board, CIO and operational teams can move from impact assessment to practical improvement.

Executive position

Sportetitive suffered a serious ransomware event after attackers used social engineering against a managed service provider, captured privileged credentials, moved across the environment and later encrypted systems while stealing personal data. The attack succeeded because several weaknesses aligned at the same time, including missing MFA, broad administrator access, supplier over-trust, weak early detection and the limitations of a legacy POS environment.

This means the response cannot be reduced to technical clean-up. The organisation needs a board-led cyber resilience programme that connects business continuity, legal accountability, supplier oversight, backup assurance, staff awareness and long-term technology investment.

Website purpose

This site is organised into focused sections so that leadership can review the attack path, the control strategy, the incident response model and the CIO-led implementation roadmap without reading the full dissertation material first.

  • Mind-map view of preparation, detection, containment, eradication, recovery and lessons learned.
  • MITRE ATT&CK and Zero Trust mapping that shows where the attack could have been interrupted.
  • A phased CIO delivery plan that links governance, technology, supplier assurance and board reporting.

Confirmed attack path

01

Attackers gathered public information from LinkedIn and supplier relationships to create a convincing pretext.

02

A spoofed CEO-style email directed the MSP technician to a fake survey link, leading to malware installation.

03

Keylogging activity captured privileged credentials, giving the attackers a route into core systems and cloud services.

04

The attackers moved laterally, identified valuable data, disabled protections and prepared the ransomware impact stage.

05

Web services failed, files were encrypted and personal data exposure was later confirmed, creating both operational and regulatory consequences.

06

Recovery then depended on board decisions, evidence preservation, insurer engagement, legal assessment and restoration from clean backups.

Free hosting recommendation

The best free option for this website is GitHub Pages if you want the simplest setup for a static HTML, CSS and JavaScript site. GitHub Pages is free for public repositories, supports custom domains, free HTTPS and Git-based deployment, and is especially suitable for academic project sites, portfolios and documentation-style pages.

If you want a slightly more flexible deployment workflow later, Netlify is also strong, but GitHub Pages is the easiest starting point for this exact website because it is fully static and does not need a server.
